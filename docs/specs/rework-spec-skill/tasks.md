# Tasks: Rework Spec Skill

## Goal

Rework `/spec` and `/next-task` into a two-tier, TDD-driven workflow with no requirements.md, no micro tier, and domain-agnostic templates. The result ships as a plugin to `li-productivity-agents/linkedin-plugins/infrastructure/online-analytics/skills/`.

**Acceptance Criteria:**
- `/spec` generates only two tiers: standard (tasks.md + next-step.md) and full (design.md + tasks.md + next-step.md)
- No references to requirements.md, micro.md, spec.md, or EARS format remain in any skill file
- Templates contain structural skeletons only — no REST, JWT, TypeScript, web-app phases, or other domain-specific content
- `/next-task` uses plan mode as entry point and TDD execution (test → implement → verify)
- Template paths are relative links, not hardcoded `~/.claude/` paths
